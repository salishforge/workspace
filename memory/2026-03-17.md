### 2. Multi-Model Support Strategy (CRITICAL DECISION)
- **Problem:** Tidepool is Claude-only; blocks Gemini, GPT-4, multi-model flexibility
- **Initial analysis:** Three paths (patch Tidepool, migrate to AutoGen, hybrid)
- **Realization:** MemForge + NATS are framework-agnostic; extract them as standalone services
- **New strategy:** Build Salish Forge as **platform infrastructure**, not framework-specific

### 3. Standalone Platform Services (APPROVED)
Two new independent services:

**MemForge Service** (`salishforge/memforge`)
- Extracted from Tidepool as reusable npm module
- REST API + SDK
- Multi-tenant (per-agent memory isolation)
- Hot/warm/cold tier management
- Autonomous consolidation
- Can plug into AutoGen, Tidepool, CrewAI, others
- Effort: 1-2 weeks extraction + docs

**Hyphae Service** (`salishforge/hyphae`)
- Named for underground fungal network (mycelium metaphor)
- Not just message bus; full **federation layer**
- Service registry + discovery
- Protocol translation (AutoGen ↔ Tidepool ↔ OpenClaw)
- Load balancing across service instances
- Capability enforcement + audit logging
- Enables multi-platform agent teams
- Effort: 3-4 weeks core + 2 weeks adapters

### 4. Hyphae Architecture (Federation Layer)
Instead of loose services, Hyphae becomes:
- Central orchestration platform
- Service registry (what services exist)
- Service discovery (automatic)
- Routing (message + service calls)
- Protocol translation (client doesn't know platform)
- Load balancing (multiple MemForge instances)
- Composition (multi-step workflows)
- Audit (unified log)

**Result:** AutoGen agents + Tidepool agents + OpenClaw agents all in one network, sharing services transparently.

### 5. Stabilization Before Multi-Agent Control (CRITICAL)
- **John's insight:** Can't safely coordinate two autonomous agents (John + Clio) on experimental infrastructure
- **Decision:** Stabilize Tidepool first (3-4 weeks)
- **Requirements before Clio involvement:**
  - Agent execution bulletproof (race conditions fixed)
  - NATS production-ready (NKeys + TLS + ACLs deployed)
  - MemForge consolidation proven (72-hour test)
  - Audit logging complete (all operations logged)
  - Error recovery documented
  - 24-hour continuous operation test passing
- **Acceptance criteria:** Can trace any message through system; understand any failure; recover from any transient failure

### 6. Infrastructure-First Philosophy (Confirmed)
- Salish Forge is not a framework choice; it's a **platform layer**
- MemForge = memory infrastructure
- Hyphae = coordination infrastructure
- Frameworks (Tidepool, AutoGen, etc.) = clients of the platform
- This gives optionality: can migrate frameworks without losing memory system
