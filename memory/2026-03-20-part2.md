### Implementation Details

**Key Design Decisions:**

1. **File-based loading:** Policies loaded from disk on each request, not cached
   - Pro: Changes immediately visible
   - Pro: No restart needed
   - Pro: Simple to understand
   - Con: Slight file I/O per request (minimal)

2. **History tracking:** All changes stored in same JSON file
   - Pro: Single source of truth
   - Pro: Audit trail included
   - Pro: Easy to export/backup
   - Con: File grows over time (100-entry limit mitigates)

3. **Rollback logic:** Uses history to find previous state
   - Pro: Non-destructive (creates new history entry)
   - Pro: Can rollback multiple times
   - Pro: Full audit trail
   - Con: Requires at least 2 entries for rollback

4. **Dashboard integration:** Prepared but not yet connected
   - Save buttons use alerts (placeholder)
   - History modal uses mock data (placeholder)
   - Ready for RPC integration in next phase
