## Meta: Effort Estimation Calibration (2026-03-16)

**Issue logged by John:** All effort estimates use human development time as baseline. AI coding agents (Flint/Clio) routinely complete "2-day" estimates in under an hour. This creates planning dysfunction — sprint scoping, resource allocation, and timeline commitments are all wrong by an order of magnitude.

**Proposed fix:** Build a calibration dataset:
1. Define a set of canonical tasks with pre-estimates (use human-hours as the unit)
2. Complete each task and record actual agent wall-clock time
3. Derive a multiplier (e.g., "1 human-day = ~45 agent-minutes" or per-task-type modifiers)
4. Apply multiplier to all future estimates

**Action item:** Track Sprint 8 task completion times against estimates as the first calibration dataset. Log start/end timestamps per task. Review after sprint close.

**Note:** Modifier will likely vary by task type:
- Pure code generation: very fast (10-30x)
- Code + debugging + iteration: fast (5-10x)  
- Architecture + design decisions: near-parity (1-2x) — constrained by human input loops
- Infrastructure/ops with human approval gates: near-parity or slower
