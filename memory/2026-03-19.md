### Six Core Agentic Design Patterns (Standardized)
1. **Reflection** — Agent observes own outputs, self-corrects
2. **Tool Use** — Structured API integration
3. **Planning** — Break complex tasks into sequences
4. **Multi-Agent Collaboration** — Multiple agents on same problem
5. **Orchestrator-Worker** — Hub-and-spoke pattern (Flint + Clio architecture)
6. **Evaluator-Optimizer** — Separate agent for quality feedback

### Reasoning Patterns (Research-Validated)
- **ReAct (Reason + Act)** — Interleave reasoning & action at each step (proven best by Yang et al. 2023)
- **Chain-of-Thought** — Explicit reasoning before action
- **Tree-of-Thoughts** — Explore multiple reasoning paths in parallel

### Multi-Agent Orchestration (Just Standardized)
- **Microsoft Agent Framework** (released Dec 2025) — Latest evolution
- **Sequential, Concurrent, Group Chat, Handoff, Magentic** patterns
- **Hub-and-spoke** architecture (central orchestrator + specialized workers)
- **Google ADK** — Multi-agent pattern abstractions
- **Amazon Bedrock** — Memory layer for conversational continuity

### Prompt Engineering Best Practices (Disciplined Approach)
- 11 proven techniques (Augment Code, 2025)
- Structured format: role, capabilities, constraints, tool definitions, error handling
- Model-specific variants (Claude ≠ GPT-5 ≠ Gemini)
- Iterative empirical improvement (not cargo-cult patterns)

### Validation Framework (2026 Standard)
- **Behavioral validation > syntactic checking** — New industry standard
- **50%+ test-to-code ratio** — AI-generated code requires extensive testing
- **Requirements agent pattern** — Verify output against acceptance criteria
- **Outcome-based validation** — "Did we achieve the goal?" not "Is this line correct?"

### Memory & State Management (Non-Negotiable)
- **LLMs are stateless** — External memory layer is critical for production
- **Session memory** (short-term context, minutes-hours)
- **Long-term persistent memory** (knowledge persistence, days-indefinitely)
- **LangGraph checkpointing** (proven pattern)
- **Mem0, Bedrock, Microsoft Foundry** — Industry solutions converged
- **Research consensus:** Structured persistent memory critical for long-term coherence

### What's Being Built Now (Dec 2025 - Jan 2026)
- Microsoft Foundry Agent Service with managed memory (Dec 2025)
- Google ADK with multi-agent patterns (Dec 2025)
- Amazon Bedrock AgentCore memory consolidation (Oct 2025)

### What's NOT Standardized Yet
- Version control for AI-generated code (experimental)
- Standardized safety validation (each org custom)
- Cost optimization frameworks (limited guidance)
- Model routing architectures (Microsoft adding)
- Formal agent SLAs (not yet production-standardized)

### Key Industry Consensus (Jan 2026)
1. **Multiple agents are necessary** — Single agent insufficient
2. **State persistence is critical** — Stateless agents fail in production
3. **Structured prompting works** — Ad-hoc prompting unreliable
4. **Validation replaces review** — Behavioral tests > code inspection
5. **Orchestration is hard** — Agents easy, coordination hard
6. **Memory is competitive advantage** — Persistent state = better agents
