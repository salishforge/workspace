## Architecture Decisions Made (John's expectations anticipated)

### 1. Service Registry vs. Core
**Decision:** Separate services (3100 = core routing, 3108 = service registry)
**Rationale:** Clear separation (discovery vs. provisioning), scales independently

### 2. Credential Strategy
**Decision:** Encrypt at rest, show only once, use directly with services
**Rationale:** Minimal attack surface, direct service communication for performance

### 3. Hyphae in Data Path?
**Decision:** NO - agent uses service directly after provisioning
**Rationale:** Hyphae controls via secrets, audit via encrypted logs, not in message path

### 4. Policy Model
**Decision:** Per-agent per-service with temporal validity
**Rationale:** Flexible authorization, time-limited access, easy to audit

### 5. Encryption Standard
**Decision:** AES-256-GCM + PBKDF2 (100k iterations)
**Rationale:** NIST-approved, defense in depth, resistant to timing attacks
