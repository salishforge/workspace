## Design Phase (13:00–14:30 PDT)

Designed production-ready Model Router Service with complete specification:

**Service Architecture:**
- Service Registry (8 pre-configured services)
- API Key Management (AES-256-GCM encryption, approval workflow)
- Intelligent Router Algorithm (task classification + 7-factor scoring)
- Cost & Limit Manager (daily/monthly/rolling tracking, alerts at 70%, hard stop at 100%)
- Admin Dashboard (web UI for approvals, usage, costs)
- Push Notifications (Telegram alerts for admin)

**Database Schema:**
- `hyphae_model_services` (8 services pre-registered)
- `hyphae_model_api_keys` (approval workflow + encryption)
- `hyphae_model_limits` (per-agent limit tracking)
- `hyphae_model_usage_log` (immutable usage audit trail)
- `hyphae_model_audit_log` (write-only with triggers)

**Routing Algorithm Factors:**
1. Policy compliance (hard constraints)
2. Budget/limit status (availability filtering)
3. Cost per token (efficiency scoring)
4. Task fit (coding→Claude Max/Opus, chat→Flash/Haiku)
5. Reset timing (prefer long windows)
6. Urgency (prioritize Claude Max for urgent)
7. Limit headroom (prefer unused services)

- Adjusted cost estimates accordingly
- Removed local GPU assumptions

**Decision 2: Service Registry Approach**
- Chose unified service registry with pre-registration
- Supports dynamic service discovery
- Clean separation of concerns
- Ready for future service additions

**Decision 3: Approval Workflow**
- Push notifications to admin for approval
- Configurable policies (auto-approve under budget, require approval for expensive)
- Per-agent tracking
- Full audit trail

**Decision 4: Intelligent Routing**
- 7-factor scoring system chosen over simple rules
- Balances cost optimization with policy enforcement
- Ready for ML-based improvements later
- Transparent scoring (can audit decisions)

**Decision 5: ES Module Migration**
- Project uses ES modules by default
- All service code converted from CommonJS
- Maintains compatibility with Hyphae core
- Future-proofs for Node.js upgrades

## Lessons Learned

1. **Module System Consistency:** Verify project-wide module system before writing code
2. **Schema Validation:** Test schema separately before integrating into service startup
3. **Dependency Management:** Check npm dependencies early (node-fetch)
4. **PostgreSQL Compatibility:** Index syntax varies by version (test with target version)
5. **ES Module Exports:** Requires explicit export statements (not module.exports)
