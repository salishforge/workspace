## Architecture Decision

**Final Design:** NATS Message Bus + MCP Knowledge Layer
- **NATS (sf.agent.{name}.*)** — High-speed coordination, request/reply, pub/sub
- **MCP (100.97.161.7:8484)** — Memory queries, tool invocation, artifact sharing
- **Bridge Scripts** — Node.js services subscripting to NATS topics, writing to workspace trigger files
- **Audit Log** — PostgreSQL table logging all inter-agent traffic

**Why NATS over MCP-only:**
- Proven need: This week's inotifywait+SSH+Telegram duct-tape pipeline proved pub/sub is essential
- Request/reply pattern exactly matches agent coordination needs
- Scales cleanly to 3+ agents without mesh complexity
- Light infrastructure (20MB RAM)

## Key Security Decisions

1. **NKeys Auth** (day-one requirement per Clio)
   - Ed25519 keypair per agent (not custom signing)
   - Operator account model (org-level trust)
   - Per-topic ACLs (what each agent can publish/subscribe)

2. **Capability Whitelists**
   - Receiving agent validates incoming requests against sender's authorized capabilities
   - Prevents prompt injection propagation even if sender is compromised

3. **Audit Logging**
   - Every NATS message → PostgreSQL (timestamp, sender, recipient, action, hash, security_result)
   - Non-negotiable for trust and debugging

4. **Circuit Breakers**
   - Max 10 messages/minute per agent pair
   - Max conversation depth: 5 rounds before human escalation
   - Dead letter queue for failed messages

## Lessons Learned

1. **Deployment & Coordination:** Trying to touch VPS services without explicit Clio coordination is risky. The NKeys setup is correctly sequenced to require her input first.

2. **Separation of Concerns:** The architecture cleanly separates:
   - **NATS:** Real-time messaging & coordination
   - **MCP:** Knowledge & tools (unchanged)
   - **PostgreSQL:** Audit trail (new)
   - **Docker:** Sub-agent isolation (policy enforcement)

3. **Security from Day One:** NKeys, ACLs, audit logging, circuit breakers are not optional. They're foundational.

4. **Documentation Matters:** Capability matrix, container spec, and security policy docs are as important as the code.
