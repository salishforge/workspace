### Near-term (v1.3.0)
1. **Redis Clustering** — If >100 agents causes cache saturation, evaluate Redis Cluster or read replicas
2. **DB Connection Pool Tuning** — Profile `pg.Pool` `max` setting against observed peak connections
3. **Token Caching** — Pre-warm token cache for batch agent startups to reduce cold-start burst

### Scaling Limits (projected)
Based on v1.1.0 performance (7,500 req/sec capacity, <0.02% error rate):
- **Current capacity:** ~150–200 agents (conservative estimate)
- **Bottleneck at scale:** PostgreSQL full-text search (warm tier queries)
- **Next scaling lever:** pgvector semantic search + read replicas
- **10,000 req/sec target:** Requires Redis Cluster + horizontal MemForge instances

# 2. Create test OAuth2 client (if not exists)
